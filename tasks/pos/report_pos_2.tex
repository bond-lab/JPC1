\section{Introduction}
\label{sec:introduction}

The main goal of this work is to extend already established metaphorical relationships to new lexical forms, with the intention of expanding them across different parts of speech, though not limited to this. Rather than treating metaphors as fixed expressions, this work approaches them as relationships that can be further examined to reveal additional metaphorical connections. 

\section{Background}
\label{sec:background}

This work builds on existing lexical-semantic resources, particularly WordNet, which provides structured information about word meanings, synonyms, derivations, definitions, and example sentences. These resources enable systematic exploration of semantic and metaphorical relationships between words.

\section{Approach}
\label{sec:approach}

The program is designed to expand well-known metaphorical pairs such as \textit{illuminate--understand} to other related lexical forms, potentially across different parts of speech. For example, it can generate a pair such as \textit{clarity--understanding}, where, unlike the original pair, both words are nouns. In our study, the first word (clarity) is treated as a metaphorical expression, while the second (understand) represents the source concept from which the metaphorical meaning of clarity is derived. Moreover, the program attempts to estimate the exact sense of the source concept.

The core of the program is the function \texttt{extend\_metaphor}, which performs most of the analysis for each pair. It begins by calling \texttt{word\_related\_forms} to collect all synonyms and derivationally related forms for both the metaphorical word and the source word (e.g., dark $\rightarrow$ darkness). For every metaphorical form, the program checks all source forms against all definitions and examples of that form. Whenever a source form appears in a definition or example, the \texttt{highlight} function marks it in the text. Highlighting not only visually identifies the source word but also provides input for \texttt{sense\_key\_from\_marked\_words}. This function uses the highlighted sentence to apply Word Sense Disambiguation using the Lesk algorithm, estimating the exact meaning (synset) of each highlighted word and returning its sense key (lemma + synset).

Each match found by \texttt{extend\_metaphor} is stored as a structured entry in the results set, including:

\begin{itemize}
\item the metaphorical word
\item its part of speech
\item the type of match (definition or example)
\item the metaphor’s sense key (metaphor lemma + metaphor synset)
\item the highlighted sentence
\item the sense keys of the estimated source words (source lemma + source synset)
\end{itemize}

Finally, the program iterates through a predefined list of metaphor--source pairs in \texttt{main}. For each pair, \texttt{extend\_metaphor} produces an organized overview of how abstract concepts are expressed metaphorically across different lexical forms.

Additionally, the program calculates the distances between the senses (synsets) of the metaphorical and source words using the \texttt{sense\_key\_distance} function. However, such distances can only be computed when both words share the same part of speech, since WordNet does not define semantic distance between synsets belonging to different parts of speech.

\section{Discussion}
\label{sec:discussion}

\subsection*{Correct Matches}

Although the distance between senses cannot be computed for most pairs (see the last piece of information in each data item), because the metaphorical and source words belong to different parts of speech, the large number of matches found still represent legitimate extensions of the original metaphorical pairs.

In some cases, additional verification was necessary to determine whether the metaphorical word is etymologically related to the source concept. For example, for the metaphorical pair \textit{fire} -- \textit{passion}, the program produced many matches, including: \textit{ardor}, \textit{ardour}, \textit{fervency}, \textit{fervent}, \textit{fervid}, \textit{fervidness}, \textit{fervor}, and \textit{fervour}. These forms needed to be verified as historically related to the concept of fire. Etymological evidence confirms this connection. The words derive from Latin \textit{ardor}, meaning: “burning heat, flame”. Several related forms are connected to Latin \textit{ferve\=o}, meaning: “to boil, to be hot, to glow”.

A similar pattern appears in the metaphorical pair \textit{cold} -- \textit{unfriendly}. For example, the word \textit{frigid} derives from Latin \textit{fr\={\i}gus}, meaning: “coldness, frost”.

Another example is the pair \textit{death} -- \textit{end}. The word \textit{expiry} derives from Latin \textit{exp\={\i}r\=are}, meaning: “to breathe out, blow out, exhale; to breathe one’s last, die”.

Sometimes, because of the intertwining of the pair, both words actually relate to the metaphor. For the pair \textit{fire}--\textit{passion} word \textit{ardour} is defined as “feelings of great \textless warmth\textgreater{} and intensity”. While \textit{warmth} relates to fire, the context of the description indicates that it refers to \textit{feelings} and \textit{intensity}, so the word is verified to be a correct match.

Another example of intertwining occurs in the special case of the pair \textit{death}--\textit{end}, where \textit{last}--\textit{death} match is found, which is correct, although in the reverse direction.

\subsection*{Incorrect Matches}

Incorrect matches arise from several distinct causes.  

A frequent cause is when terms are often used interchangeably. For example, pair \textit{death}--\textit{end}  yields many false matches simply because the same word appears in a sentence example of its match, e.g., “death - example match: her \textless death\textgreater{} came as a terrible shock” or “end - example match: the \textless end\textgreater{}  was exciting.” 

Another source of error is when words are related only as synonyms, without any true etymological connection. For instance, the pair \textit{fire}--\textit{anger} may match “provocation - definition match: unfriendly behavior that causes \textless anger\textgreater{} or resentment”.

False matches can also occur by coincidence. For example, source word \textit{understand} appears in an example sentence like “lighting - example match: an interior decorator must \textless understand\textgreater{} lighting” where the connection to the metaphorical word is incidental.  

Problems also arise when synonyms of a source word correspond to highly common words. For the source \textit{understand}, words like \textit{read} or \textit{see} may generate many matches that are not meaningful.  

Finally, a failure such as “no results found” sometimes happens simply because we miss the data to match any pair forms.

In summary, incorrect matches can result from interchangeable terms, synonymous associations, coincidental occurrences, overly common forms, or missing data.

\section{Conclusions}
\label{sec:conclusions}

We examined how familiar metaphors can appear in forms different from their conventional expressions. This project provides insight into the flexible use of metaphor in language and shows that computational methods can help automatically identify metaphoric relationships.
